# Qiita Review Rules

## Scope

Review only the article content intended for Qiita.

Focus on:

- Article quality
- Readability
- Markdown quality
- Basic proofreading
- Obvious technical inaccuracies
- Rendered preview quality

Do not review:

- Rust source code
- Workflow implementation details
- Repository governance
- Merge readiness

## Severity

Allowed labels:

- `[suggestion]`
- `[warning]`

Do not use:

- `[critical]`
- Blocking language
- Required-change language

## Output

Keep feedback concise and concrete.

Limits:

- Maximum 3 suggestions
- Maximum 2 warnings
- Include positive feedback when possible

Prefer small, actionable comments over rewrite requests.
